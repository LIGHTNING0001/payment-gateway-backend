# 第四方支付网关对接文档

## 1. 项目概述

本项目是一个第四方支付网关，提供统一的支付接口，支持对接多种支付通道（如支付宝、微信支付等）。通过本网关，商户可以使用统一的接口进行支付、退款、查询等操作，无需直接对接各个支付通道，简化了对接流程。

## 2. 接口列表

| 接口名称 | 接口路径 | 方法 | 功能描述 |
|---------|---------|------|----------|
| 统一收单/支付接口 | /api/v1/pay/create | POST | 接收支付请求，调用上游通道下单，返回支付链接或二维码 |
| 订单查询接口 | /api/v1/pay/query | POST | 查询某笔订单的支付状态 |
| 退款申请接口 | /api/v1/pay/refund | POST | 处理商户的退款请求 |
| 退款查询接口 | /api/v1/pay/refund/query | POST | 查询退款的处理进度和状态 |
| 异步通知回调接口 | /api/v1/pay/notify | POST | 接收上游通道的异步支付成功通知 |
| 支付方式列表接口 | /api/v1/pay/methods | GET | 动态获取当前商户可用的支付通道和费率信息 |
| 商户列表数据 | /api/v1/merchant/lists | GET | 返回商户号_环境格式的商户列表 |
| 预下单接口 | /api/v1/pay/precreate | POST | 返回请求参数，供前端对接使用 |
| CFCA加密接口 | /api/v1/security/cfca/encrypt | POST | 使用CFCA证书加密数据 |
| CFCA解密接口 | /api/v1/security/cfca/decrypt | POST | 使用CFCA证书解密数据 |
| RSA2加签接口 | /api/v1/security/rsa2/sign | POST | 使用RSA2算法对数据进行加签 |
| RSA2验签接口 | /api/v1/security/rsa2/verify | POST | 使用RSA2算法对数据进行验签 |

## 3. 接口详细说明

### 3.1 统一收单/支付接口

**接口路径**: `/api/v1/pay/create`

**请求方法**: POST

**请求参数**:

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| merchantNo | String | 是 | 商户号 |
| signType | String | 是 | 签名类型，支持RSA2和CFCA |
| sign | String | 是 | 签名 |
| signContent | String | 是 | 签名内容，JSON格式 |

**signContent参数**:

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| transNo | String | 是 | 交易号 |
| payType | String | 是 | 支付类型，如ALI_NATIVE、WECHAT_NATIVE等 |
| orderAmt | String | 是 | 订单金额，单位：元 |
| totalOrderAmt | String | 是 | 总订单金额，单位：元 |
| goodsInfo | String | 是 | 商品信息 |
| requestDate | String | 是 | 请求日期，格式：yyyyMMddHHmmss |
| returnUrl | String | 是 | 支付完成后返回的地址 |
| memo | Object | 否 | 备注信息，JSON格式 |

**响应参数**:

| 参数名 | 类型 | 描述 |
|--------|------|------|
| transNo | String | 交易号 |
| orderAmt | BigDecimal | 订单金额 |
| qrCode | String | 支付二维码链接 |
| status | String | 订单状态，0：待支付，1：支付成功，2：支付失败，3：已退款 |

**示例请求**:

```json
{
  "merchantNo": "891065321967",
  "signType": "RSA2",
  "sign": "签名值",
  "signContent": "{\"transNo\":\"2026032812000001\",\"payType\":\"ALI_NATIVE\",\"orderAmt\":\"1\",\"totalOrderAmt\":\"1\",\"goodsInfo\":\"测试商品\",\"requestDate\":\"20260328120000\",\"returnUrl\":\"https://example.com/return\",\"memo\":{}}"
}
```

**示例响应**:

```json
{
  "transNo": "2026032812000001",
  "orderAmt": 1,
  "qrCode": "https://qr.alipay.com/bax00000000000000",
  "status": "0"
}
```

## 4. 签名生成和验签说明

### 4.1 RSA2签名

1. 使用商户的私钥对signContent进行SHA256WithRSA签名
2. 将签名结果转换为小写的十六进制字符串
3. 将签名值放在sign参数中

### 4.2 CFCA签名

1. 使用商户的CFCA私钥对signContent进行SHA1WithRSA签名
2. 将签名结果转换为小写的十六进制字符串
3. 将签名值放在sign参数中

### 4.3 验签

1. 从请求中获取signType、sign和signContent
2. 根据signType选择对应的验签算法
3. 使用商户的公钥对signContent和sign进行验签
4. 验签成功后才处理请求

## 5. 错误码说明

| 错误码 | 错误信息 | 描述 |
|--------|----------|------|
| 500 | 验签失败 | 签名验证失败，可能是签名不正确或签名过期 |
| 500 | 原订单不存在 | 退款时原订单不存在 |
| 500 | 系统内部错误 | 系统内部出现错误，请联系技术支持 |

## 6. 对接流程

1. 商户在支付网关注册，获取商户号和密钥
2. 商户根据文档准备请求参数，生成签名
3. 商户调用统一收单/支付接口，获取支付二维码或链接
4. 用户扫描二维码或点击链接完成支付
5. 支付通道异步通知支付网关支付结果
6. 支付网关更新订单状态，并异步通知商户支付结果
7. 商户可通过订单查询接口查询订单状态
8. 如需退款，商户调用退款申请接口
9. 支付网关处理退款请求，调用支付通道的退款接口
10. 商户可通过退款查询接口查询退款状态

## 7. 注意事项

1. 商户需要妥善保管私钥，避免泄露
2. 签名生成和验签必须使用正确的算法和密钥
3. 交易号必须唯一，建议使用时间戳和随机数生成
4. 订单金额必须精确到小数点后两位
5. 异步通知回调接口必须能够正确处理重复通知
6. 商户应及时查询订单状态，确保支付结果的准确性
